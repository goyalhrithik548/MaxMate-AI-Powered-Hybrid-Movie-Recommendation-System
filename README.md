## ğŸ” Project Overview

In the current streaming landscape, users face "decision fatigue." Traditional keyword searches fail to understand nuanced human intent (e.g., *"I want a scary movie that has a happy ending"*).

**MaxMate** bridges this gap by acting as an intelligent "Virtual Concierge." It combines traditional machine learning with modern Generative AI to provide accurate, personalized, and conversational recommendations without sending user data to the cloud.

---

## âœ¨ Key Features

### 1. Hybrid Recommendation Engine
* **Engine A (Demographic Filtering):** Solves the "Cold Start" problem for new users by ranking movies using the **IMDB Weighted Rating** formula.
* **Engine B (Content-Based Filtering):** Personalizes results using **Cosine Similarity** on "Bag of Words" vectors (Cast, Director, Keywords, Genres).

### 2. Generative AI Chatbot ('Max')
* **RAG Pipeline:** Uses **Retrieval-Augmented Generation** to fetch relevant movie plots from the dataset and feed them to the LLM.
* **Local Inference:** Powered by a quantized **Microsoft Phi-2** model running entirely on the CPU.
* **No Hallucinations:** The AI is strictly constrained to answer based only on the retrieved context.

### 3. Smart User Interaction
* **Implicit Feedback:** Tracks user engagement (if a user views a movie detail page for >15 seconds).
* **Explicit Feedback:** "Like" and "Save to Watchlist" functionality.
* **Vector Search:** Semantic search capabilities beyond simple keyword matching.

## ğŸ— System Architecture

The system follows a modular **Client-Server Architecture**:

1.  **Frontend:** HTML5, CSS3, JavaScript (AJAX for asynchronous calls).
2.  **Backend:** Flask (Python) handles routing, session management, and API endpoints.
3.  **Database:** MongoDB stores user profiles, hashed passwords, and interaction history.
4.  **AI Engine:**
    * **Memory:** Pre-computed Pickle Artifacts (`.pkl`).
    * **Brain:** `llama-cpp-python` (LLM) and `sentence-transformers` (Embeddings).
